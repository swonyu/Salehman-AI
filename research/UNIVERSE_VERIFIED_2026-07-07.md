# Universe expansion — 2000 verified US equities (2026-07-07)

**What:** Added 2000 US-listed stocks to the searchable catalog (`StockSageQuoteService.catalogExtra`),
growing the app's universe from ~210 analyzed-core / ~390 total to ~2390 searchable names.

## Method (honesty floor: every symbol live-sourced + app-path-verified, none from model memory)
1. **Sourced LIVE** from the Nasdaq screener API (`api.nasdaq.com/api/screener/stocks`, 7,176 rows,
   as-of 2026-07-07) — real, currently-listed US common stock with market cap, sector, volume.
2. **Ranked** by market cap (liquidity-first); normalized to Yahoo symbol convention (`BRK/B`→`BRK-B`);
   dropped warrants/units/odd tickers; **deduped** against the existing 390 core symbols.
3. **VERIFIED every candidate fetchable through the app's own Yahoo v8 path** (`query1.finance.yahoo.com/v8/finance/chart`),
   gentle (concurrency 4, backoff, abort-on-sustained-429): **2001 verified fetchable with ≥5 recent bars, 6 dropped, 0 throttling.**
   Spot-checked data freshness: all sampled symbols returned 2026-07-06 closes (matching AAPL).

## Placement decision (why catalog, not the analyzed board)
The **analyzed core** (`worldwide`, auto-fetched every refresh) stays curated at ~210 — fetching 2000/refresh
at concurrency 6 would take ~5+ min, blowing the 120s watchdog and re-triggering the Yahoo throttle. The 2000
went to the **searchable catalog** (add-on-demand, per the existing London+/Frankfurt+ long-tail pattern): each
is searchable + addable, fully analyzed when a user adds it. A bigger AUTO-ANALYZED board is the owner-gated
pipeline-rework project (re-budget watchdog + batched fetch + offline history cache) — NOT done here.
