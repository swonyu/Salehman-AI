# Concurrency / await-interleaving bughunt (wafc1ymeh, 2026-06-22)

3 CONFIRMED await-suspension interleavings (MainActor does NOT protect — cross-method inconsistent snapshots / missing cancellation). #1 (remove-during-refresh reverts + re-caches) DONE. RE-VERIFY vs source.

### ✅ DONE #1 [medium] — removeSymbol() during an in-flight refresh() is reverted on screen AND re-persisted to the disk cache — userSymbols and the saved board diverge, surviving relaunch
**fix:** refresh() never re-checks the live userSymbols set after its await, so a removal that lands mid-fetch is undone. Interleaving: (1) FOO is in userSymbols; (2) refresh() captures universe=trackedDefs() at 626 (includes FOO) and suspends at 627, FOO in the fetch list; (3) user taps remove -> removeSymbol("FOO") completes on the main actor, deleting FOO from userSymbols (147), shortening UserDefaults (148), removing the FOO row from symbols (149); (4) refresh() resumes: quotes["FOO"] priced -> line 632 rebuilds FOO into live; preservedUserRows (656) is computed from post-removal symbols (FOO absent) but FOO is already back via live, so committed=live+preservedUserRows (659) contains FOO; replaceAll(committed) (660) puts the just-removed FOO back on the board and line 666 saves committed (incl. FOO) into salehman_quote_cache.json. Wrong state: removal reverted on screen; cache now a superset of userSymbols; loadCachedQuotes() (83-91) rebuilds symbols purely from cache and never reconciles against userSymbols, so FOO reappears on next launch too. FIX: after the await, reconcile the commit set against the CURRENT tracked set before building committed. At ~655 compute let tracked = Set(userSymbols.map { $0.uppercased() }) and filter user-market rows that are no longer tracked: let liveFiltered = live.filter { $0.market != Self.userMarketLabel || tracked.contains($0.symbol.uppercased()) }; use liveFiltered for both committed and the cache save (curated rows unaffected since removeSymbol no-ops for them). This keeps the saved cache equal to what is shown and honors the mid-flight removal.

### ⬜ #2 [low] — StockSageMonitor.stop() does not interrupt the in-flight cycle — alerts fire after monitoring is turned off, and a quick stop->start double-fires notifications and races lastAlerted
**fix:** runCycle() has no Task.isCancelled check anywhere and the loop never re-checks cancellation between await refresh() (49) and await runCycle() (50). Interleaving: the loop Task is suspended at await sendAlert (90) inside runCycle's for-loop when the user toggles monitoring off; stop() (61-65) cancels and sets isRunning=false, but execution resumes at 90, finishes iterating the snapshot, fires the remaining sendAlert notifications, and writes lastAlerted (98) AFTER the user disabled alerts. Worse, if the user immediately toggles back on, toggleMonitoring (MarketsView.swift:528) calls start(), whose guard !isRunning passes (stop already cleared isRunning), spinning up a SECOND loop Task; the zombie tail of the old cycle and the new loop's first runCycle(notify:true) both call sendAlert (90) and both mutate the shared lastAlerted dictionary (98) interleaved across their await suspension points -> duplicate notifications for the same symbol and a dedupe map written by two overlapping cycles. (checkAlertsNow at MarketsView.swift:537 runs notify:false, so line 98 is guarded out -> not a writer; correctly excluded.) FIX: honor cooperative cancellation so stop() actually stops the cycle: (1) guard !Task.isCancelled else { return strong } at the top of runCycle's for-body (~84) and again right before await sendAlert (90); (2) in the loop, check cancellation after await refresh() and after await runCycle() (49-50). Then no sendAlert is dispatched and lastAlerted is not mutated by a dying cycle, killing the post-stop notification and the stop->start double-fire / lastAlerted race.

### ⬜ #3 [low] — retryFailedIdeas() re-reads trackedDefs() AFTER its await, so a ticker added during the fetch is falsely listed under the 'couldn't be fetched' banner with a Retry button while it is priced and on the board
**fix:** This is a cross-method inconsistent-snapshot, not a low-level race (the store is @MainActor and the re-entry guard at 199 has no await before isLoadingIdeas is set at 201). retryFailedIdeas() snapshots the universe once at 205 (defs = trackedDefs().filter{retrySet...}) BEFORE await fetchHistories (206), but RE-READS trackedDefs() FRESH at 217 to recompute ideasMissing. addSymbol() is gated by a separate flag (isAddingSymbol, 118), not isLoadingIdeas, and the add-ticker bar (MarketsView.swift:1848) is only disabled when the text is empty -- never tied to isLoadingIdeas -- so adds are allowed during a retry. Interleaving: (1) refreshIdeas leaves ideasMissing=["AAPL"]; (2) user taps Retry -> retryFailedIdeas captures retrySet={AAPL}, defs=[AAPL], suspends at 206; (3) during the await the user adds NVDA via addSymbol(), which prices and appends NVDA to userSymbols (133) and shows it on the board (140); (4) retry resumes, AAPL prices, merged/analyzed (216) lacks NVDA (never scanned); (5) line 217 calls trackedDefs() AGAIN -- which now CONTAINS NVDA -- and NVDA passes the filter (!analyzed.contains && assetClass != Index), so NVDA is written into ideasMissing. Wrong state: a freshly-added, freshly-priced ticker visibly on the board is simultaneously listed as a fetch failure under the 'couldn't be fetched' banner with a Retry affordance, and the EV-ranking honesty note claims a partial universe when it isn't. refreshIdeas() does NOT have this bug because it captures universe once at 165 and reuses it at 190. FIX: make retryFailedIdeas use ONE snapshot like refreshIdeas: let universe = trackedDefs() once at ~205, derive defs from it, and at 217 compute ideasMissing from that SAME captured universe instead of calling trackedDefs() again: ideasMissing = universe.map(\.symbol).filter { !analyzed.contains($0.uppercased()) && StockSageAllocation.assetClass($0) != "Index" }. The failure list then reflects exactly the set scanned; a ticker added mid-await is neither analyzed nor falsely reported (next full refreshIdeas picks it up cleanly).
