## Domain: core-scoring

The core-scoring domain is the deterministic signal and scoring pipeline for the StockSage engine. StockSageAdvisor is the apex: it orchestrates all indicator primitives from StockSageIndicators into a single TradeAdvice, applies the Barroso-Santa-Clara inverse-variance momentum scalar, enforces the 0.65 trend-family correlation cap (Ehsani-Linnainmaa 2022), computes ATR-scaled stops and 2:1 targets, and sizes positions via half-Kelly. StockSageIndicators is the pure math library providing all indicator primitives — its functions are total (nil on insufficient data). StockSageRegime provides a market-wide risk-on/off meta-gauge that biases sizing via sizingBias (0.25 crisis to 1.25 strong bull), sitting orthogonally to the per-symbol signal in StockSageAdvisor. StockSageSignalEngine is a separate legacy engine mapping single-day price moves to 5-way recommendation labels for the alert/monitoring and sort paths — it does NOT share StockSageAdvisor's regime or conviction logic. StockSageMultiTimeframe adds optional higher-timeframe confirmation (daily 50DMA vs real weekly 30WMA) fetched lazily for the idea detail view. StockSageExecutionTiming is a pure advisory appended to rationale in trending regimes, grounded in Lou-Polk-Skouras overnight-vs-intraday momentum research, with zero effect on scores or sizing. The domain is almost entirely pure and deterministic — the main side-effectful hook is the calibration path in StockSageAdvisor.suggestedWeight, which is only non-trivial when a StockSageConvictionCalibration is passed in from StockSageStore.buildIdeas.

**Cross-domain edges:** INPUTS: StockSageQuoteService/StockSagePriceHistory (fetched OHLCV data) feeds StockSageAdvisor.advise(history:). StockSageConvictionCalibration (fitted by StockSageConvictionCalibration module) fed into suggestedWeight by StockSageStore. StockSageExpectedValue.winProbEstimate + cryptoRiskScaler consumed inside StockSageAdvisor.suggestedWeight. TradeAdvice.Action/Regime types cross into StockSageExecutionTiming. StockSageAllocation.assetClass used in StockSageStore.buildIdeas to decide whether to pass a benchmark to advise(). OUTPUTS: TradeAdvice struct (produced by StockSageAdvisor) consumed by StockSageStore.buildIdeas (embedded in StockSageIdea), StockSageTradePlan, StockSageTodayPlan, StockSageMonitor (ideaAlertRecommendation), StockSageAlerts, StockSageRiskFlags, MarketsView (idea card + detail view). MarketRegime (produced by StockSageRegime) consumed by StockSageCapitalAllocator, StockSageExpectedValue (bannedFromTopRank, regimeAdjustedEVRankKey), MarketsView (regime card + sizingBias display + adjustedWeight for per-idea display sizing). StockSageSignal/StockSageRecommendation (from StockSageSignalEngine) consumed by StockSageMonitor, StockSageBriefingService, StockSageAlertDecision, MarketsView (sort + badge + alertSignals state). MultiTimeframeTrend (from StockSageMultiTimeframe) consumed by StockSageStore.@Published multiTimeframe dict, then by MarketsView idea detail sheet.

### StockSageAdvisor.swift
**Purpose:** The main scoring engine: combines trend (SMA crossover), momentum (6-month), MACD, RSI-under-regime, vol-adjusted momentum, 52wk-high proximity, and a three-timeframe confluence observer into a single TradeAdvice. Also owns TradeAdvice struct definition and all position-sizing math.
**Key symbols:** TradeAdvice (output struct with conviction 0-1 + timeframeAligned display badge), StockSageAdvisor.advise(closes:highs:lows:volumes:benchmarkCloses:) (main entry), suggestedWeight (half-Kelly sizing), stopTarget (ATR-scaled stop+2:1 target), varianceScalar (Barroso & Santa-Clara 2015 vol-inverse scalar), oversoldBounceIsBuyable (12-1 TSMOM gate for range bounces), stopMultiple (vol-tiered ATR multiplier), highProximityWeight/highProximityNeutralAnchor (52wk proximity constants), trendFamilyCap, riskPerTrade, maxWeight, relativeStrengthEnabled
**Inputs:** Raw OHLCV closes/highs/lows/volumes arrays (newest-last); optional benchmarkCloses for RS; optional StockSageConvictionCalibration passed only by StockSageStore.buildIdeas at runtime
**Consumers:** StockSageStore.buildIdeas (primary live call site — passes calibration + benchmark), StockSageBacktester.run (bar-by-bar, no calibration, close-only + optional OHLC), MarketsView (reads caveat, maxWeight, calls adjustedWeight for regime-adjusted display sizing), StockSageAdvisorTests + StockSageCalibratedSizingTests + StockSageMathInvariantTests + StockSageBacktestParityTests (tests)
**Invariants:** conviction field is 0-1 (min(abs(score),1.0)), NOT a probability of profit — the field comment says this explicitly. suggestedWeight is 0-maxWeight (0.20). score is unbounded internally but the trend-family is capped at trendFamilyCap (0.65) AFTER the varScalar attenuation. varScalar is attenuation-only (clamped to <=1.0 — calm must never amplify). highProximity term is long-side-only (max(0,...)), never fires in bearTrend regime, and is OUTSIDE the 0.65 trendFamilyCap. timeframeAligned is display-only badge, NEVER feeds score/conviction/sizing — gated on isBuy||isSell so Avoid/Hold cards can't show a bullish badge. advise() called with calibration=nil is byte-identical to prior behavior.
**Gotchas:** ER nil → 0, which means er >= 0.30 is false → choppy regime by default when <21 closes (this is the safe conservative path). RSI nil → 50 (dead neutral), which is safe. SMA50/SMA200 use REAL period guards (never min(count,200)) — a deliberate fix to stop a 50-bar history silently faking a 200DMA and making both SMAs identical. relativeStrengthEnabled is nonisolated(unsafe) var (default false, ablated 2026-06-27): tests can flip it on to verify the term still works, but shipped behavior is OFF. The vol-adjusted-momentum term requires highs/lows — a close-only call silently skips it. stopMultiple thresholds: vol>=0.70 → 2.5x, vol>=0.40 → 2.0x, <0.40 → 1.5x; nil vol → 2.0x default. highProximityNeutralAnchor 0.90 means only the top 10% of the 52wk range earns a contribution (max +0.010 — can push a borderline Buy to StrongBuy). 52wk proximity term DISABLED for close-only callers (highs==nil) to prevent monotone-uptrend close series inflating pth to 1.0 every day. F37 (2026-07-02): momentum-window label threshold raised from momPeriod>=100 to momPeriod>=120 — only momPeriod>=120 (~5.5 months) earns the '6-month momentum' label; shorter windows say 'momentum (N-day window)' with the actual bar count. 100 bars ≈ 4.6 months was a mislabel. Numeric signals unchanged. F34 (2026-07-02): score→action boundary asymmetry: +0.5 → .strongBuy (inclusive), −0.5 → .reduce (NOT .sell) — deliberate long-side bias documented in-code; shorts carry financing cost + unlimited-loss asymmetry, requiring a stricter signal for .sell. actionForScore() helper added for test pinning (no production behavior change). F34 test pins both boundaries.

### StockSageIndicators.swift
**Purpose:** Pure, dependency-free library of all technical indicator primitives used by the engine: SMA/EMA/RSI/MACD/ATR/ER/annualizedVol/returnOverPeriod/volAdjMomentum/TSMOM/trendOK/Donchian/highProximity/timeframeConfluence. All functions are total (nil on insufficient data, never NaN).
**Key symbols:** sma, ema, emaSeries, rsi (Wilder's, 0-100), macd (MACDValue struct with histogram), atr (Wilder's 14-period), efficiencyRatio (Kaufman ER: |net|/sum|steps|, 0=chop 1=trend), annualizedVolatility (log-returns stdev x sqrt(252), fraction NOT percent), returnOverPeriod (percent, NOT fraction), timeSeriesMomentum (12-1 TSMOM with skipRecent=21), trendOK (binary 12-1 gate), volumeConfirmation, relativeStrength, volAdjustedMomentum, donchian, isBreakout, highProximity (pth ratio 0-1 + effectiveWindow), timeframeConfluence
**Inputs:** Raw price arrays (closes newest-last), OHLC arrays for ATR/Donchian, optional volumes for volumeConfirmation, optional benchmark closes for relativeStrength
**Consumers:** StockSageAdvisor (primary — calls sma/rsi/macd/atr/efficiencyRatio/returnOverPeriod/annualizedVolatility/volAdjustedMomentum/relativeStrength/highProximity/trendOK/timeframeConfluence), StockSageRegime (sma, rsi), StockSageMultiTimeframe (sma), StockSageStore (annualizedVolatility for calibrated sizing + volRegime computation), StockSageAllocationOptimizer (annualizedVolatility), StockSageBacktester (atr for trailing stop sizing), StockSageTrailingStop (atr), StockSageVolRegime (annualizedVolatility), StockSageVolStability (annualizedVolatility), StockSageExpectedValue (efficiencyRatio, macd, returnOverPeriod for momentum quality score), StockSageAdvisorTests + StockSageIndicatorsTests + StockSageMathInvariantTests + StockSageMomentumQualityTests (tests)
**Invariants:** Every function returns nil rather than crashing or NaNing on edge cases. annualizedVolatility skips zero/negative closes from the log-return series (so a zero-price data error doesn't poison the vol). efficiencyRatio returns 0 (not nil) when noise==0 (flat tape). rsi returns 100/0 for all-up/all-down; returns 50 when both avgGain and avgLoss are 0. trendOK returns nil (not false) when <253 bars — callers must treat nil as 'unknown' not 'bearish' (the StockSageAdvisor.oversoldBounceIsBuyable uses ?? true for legacy safety). timeframeConfluence returns nil when trendOK returns nil OR when trendOK and 200DMA disagree — nil means 'unknown', never a guessed direction.
**Gotchas:** annualizedVolatility returns FRACTION (e.g. 0.20 for 20% vol) — NOT a percentage. returnOverPeriod returns PERCENT (e.g. 20 for +20%) — NOT a fraction. These two scales are mixed inside StockSageAdvisor and callers must not confuse them. timeSeriesMomentum uses skipRecent=21 to avoid the 1-month reversal (the documented anomaly the app's own TSMOM gate accounts for). volumeConfirmation and donchian/isBreakout are NOT called by the production advisor path — volumeConfirmation was parsimony-cut (2026-06-27, T2 ablation showed it worsened drawdown); donchian/isBreakout are superseded by the continuous highProximity term. Both remain in the file for tests and future use. timeframeConfluence has a TSMOM/200DMA double-check: if trendOK and price-vs-200DMA disagree, returns nil rather than picking a side — a deliberate 'unknown' sentinel to avoid contradicting itself.

### StockSageSignalEngine.swift
**Purpose:** Legacy price-momentum signal engine ported verbatim from StockSage v32 package's MarketSignalEngine. Maps a single price-vs-prior-price move (% change) to a 5-way recommendation label + confidence, using fixed %Δ thresholds. Runs alongside (not through) StockSageAdvisor.
**Key symbols:** StockSageRecommendation (strongBuy/buy/hold/sell/strongSell enum), StockSageSignal (symbol+recommendation+confidence+reason struct), StockSageSignalEngine.generateSignal(symbol:currentPrice:previousPrice:) (threshold-based), generateSignal(for:StockSageSymbol) (convenience overload)
**Inputs:** Two prices: currentPrice and previousPrice (or a StockSageSymbol with a .latest quote). No history required.
**Consumers:** StockSageMonitor (runCycle/runWatchlistCycle — generates alerts for strong buy/sell signals), StockSageBriefingService (generates bulk signals for a briefing), MarketsView (signalAlertRow display, alertSignals state, MarketSort.rank for sorting, recColor/recTextColor styling, live in-card signal badge via generateSignal(for:) at line 2486). StockSageAlertDecision + StockSageMonitor also use StockSageRecommendation as a shared type for the alert-dedup lastAlerted tracking.
**Invariants:** Function is total — invalid/zero prices return a hold with confidence 0.5 and honest reason 'No valid price to assess'. Confidence is capped at 0.92 (absChange/8, so a 7.36%+ move maxes out). Hold confidence is a flat 0.65 regardless of magnitude.
**Gotchas:** This engine is SEPARATE from StockSageAdvisor — it looks at a single day's price change only (no history, no regime, no RSI/MACD). It drives the alert/monitoring path and the sort/badge in MarketsView; StockSageAdvisor drives the idea-card trade plans. They can disagree. Confidence here is NOT conviction (0-1 directional strength) — it is an alerting confidence that scales with move size. The thresholds are hardcoded from the v32 package: >6% = strong, >2.5% = moderate, else hold. There is no calibration, no regime filter, and no half-Kelly sizing — purely a watchlist alert trigger.

### StockSageMultiTimeframe.swift
**Purpose:** Secondary higher-timeframe confirmation tool: reads a DAILY series (50DMA) and an explicitly-fetched WEEKLY series (30-week MA) and reports whether they trend in the same direction. Provides a MultiTimeframeTrend struct with a plain-English note.
**Key symbols:** MultiTimeframeTrend (daily+weekly Trend enum + aligned Bool + note String), StockSageMultiTimeframe.assess(dailyCloses:weeklyCloses:), trend(_ closes:period:) (private helper with 1% neutral band — reads Flat for indeterminate)
**Inputs:** Two separate price series: dailyCloses (close array) and weeklyCloses (close array of weekly bars, fetched separately). Both newest-last.
**Consumers:** StockSageStore (refreshMultiTimeframe — lazy on-demand fetch, stored in @Published multiTimeframe dict keyed by symbol). MarketsView (idea detail sheet: if mtf = store.multiTimeframe[symbol] — displayed at line 4513; triggers refreshMultiTimeframe on load at line 4669). Tests in StockSageMultiTimeframeTests.
**Invariants:** Uses the REAL 30-period guard on weekly closes — fewer than 30 weekly bars returns Flat (unknown), never a degraded shorter-window MA that could fake a confirmed higher-timeframe trend. The 1% neutral band prevents noisy flat-tape readings from flickering between Up/Down.
**Gotchas:** This module is SEPARATE from StockSageIndicators.timeframeConfluence: MultiTimeframe uses WEEKLY closes fetched separately (real weekly bars, less noisy) vs. timeframeConfluence which infers a long-term leg from daily closes via TSMOM. The two serve different UX surfaces: MultiTimeframe drives the on-demand detail-sheet confirmation badge; timeframeConfluence drives the inline timeframeAligned badge on the idea card. The lazy fetch in StockSageStore means multiTimeframe[symbol] may be nil until the user opens the detail view — callers must handle nil. refreshMultiTimeframe is a no-op if the symbol is already cached OR if ToolPolicy.isExternalAllowed is false (offline mode).

### StockSageRegime.swift
**Purpose:** Market-wide risk-on/risk-off meta-gauge: fuses S&P 500 trend (vs 200DMA), index RSI momentum, breadth (% of a fixed 14-symbol sample above own 200DMA), and VIX zones into a MarketRegime with a riskScore (-1 to +1) and sizingBias multiplier. Provides breadth() and adjustedWeight() as pure helpers.
**Key symbols:** MarketRegime (state+riskScore+signals+sizingBias+caveat struct), MarketRegime.State (trendingBull/trendingBear/ranging/crisis), StockSageRegime.assess(indexCloses:vix:breadthAbove200:), breadth(_ histories:) (fraction above 200DMA; nil when no eligible names), adjustedWeight(base:bias:cap:) (base * bias, re-capped), breadthSample (14 hardcoded symbols)
**Inputs:** indexCloses (e.g. S&P 500 daily history), optional vix (current VIX level — NOT a series), optional breadthAbove200 fraction (pre-computed by StockSageStore using breadth()). All optional — missing inputs simply don't vote.
**Consumers:** StockSageStore (assess() called in refreshRegime(), result stored as @Published regime: MarketRegime?, also breadth() + breadthSample used there). StockSageCapitalAllocator (adjustedWeight via regime.sizingBias). StockSageExpectedValue (bannedFromTopRank/regimeAdjustedEVRankKey use MarketRegime.State). MarketsView (regimeColor/regimeIcon + sizingBias display + adjustedWeight for idea display). StockSageRegimeTests + StockSageCapitalAllocatorTests + StockSageExpectedValueTests (tests).
**Invariants:** riskScore is clamped to [-1, +1]. sizingBias = 0.25 hardcoded for crisis (VIX>=40), otherwise formula 0.75 + riskScore*0.5 clamped to [0.40, 1.25]. adjustedWeight never returns negative (max(0,...)) and is always re-capped. breadth() excludes names with <200 bars from BOTH numerator AND denominator (doesn't penalize short-history names as 'below 200DMA' — avoids understating breadth).
**Gotchas:** breadthSample is hardcoded with a documented US-cap skew warning: 8 of 14 names are US (6 mega-cap tech), so breadth skews with Nasdaq sentiment, not the Saudi Tadawul market this app primarily targets. VIX is a point-in-time value (not a series) — the store reads it from the VIX quote's current price. crisis=true overrides the riskScore-based state bucket (VIX>=40 always forces .crisis regardless of other scores). This is MARKET-WIDE; the per-symbol vol brake is in StockSageVolRegime. The two are composable: StockSageCapitalAllocator applies sizingBias to Kelly, and StockSageVolRegime.sizingMultiplier is applied by the allocator on top.

### StockSageExecutionTiming.swift
**Purpose:** Advisory-only timing note appended to rationale for trend/momentum-regime ideas. Evidence: Lou-Polk-Skouras (JFE) shows past-return momentum premia accrue almost entirely overnight, so entering near the close is historically more edge-efficient. Strictly advisory — zero impact on score/conviction/stop/target/sizing.
**Key symbols:** StockSageExecutionTiming.sessionNote(action:regime:) -> String?, caveat (static string)
**Inputs:** TradeAdvice.Action (to filter non-actionable ideas) and TradeAdvice.Regime (to gate to trend regimes only).
**Consumers:** StockSageStore.buildIdeas only (line 462: appends '⏱ ' + sessionNote to extraNotes, which get folded into rationale via a TradeAdvice copy). Tests in StockSageExecutionTimingTests.
**Invariants:** Returns nil for .range regime (finding is specific to momentum/past-return strategies, not mean-reversion bounces), and nil for .hold/.avoid actions (non-actionable). Only fires for .buy/.strongBuy/.sell/.reduce in .bullTrend/.bearTrend regimes.
**Gotchas:** This is a ZERO-TURNOVER lever: it only notes WHEN to execute an already-decided trade, never whether to trade. It is deliberately NOT wired into score, conviction, stop, target, or suggestedWeight — the same advisory-only pattern as ReturnShape/VolStability/VolRegime/SectorRotation. The note is prefixed with ⏱ in the rationale display, distinguishing it from risk flags (⚠). The short/reduce (bearTrend) case is symmetric and also fires — momentum premia are equally documented on the short side.

## Domain: sizing-allocation

The sizing-allocation domain is the engine's "risk IS the input, size IS the output" layer: every dollar deployed flows through a deterministic pipeline that starts with Kelly fractions (StockSageKelly), passes through a filter-scale-floor orchestrator (StockSageCapitalAllocator), and terminates in whole shares (StockSagePositionSizer). A single authoritative constant — StockSageKelly.maxFraction = 0.20 — is the per-position hard ceiling imported by every sizing file. The domain enforces three nested heat controls: per-position cap (20%), portfolio scale-to-heat-cap (default 8% in allocate), and correlation de-weighting (CorrelationCluster) to prevent a "diversified" book that is actually one concentrated bet. StockSageAllocation is the orthogonal classification layer: it reads the same holdings to produce asset-class/region/sector slices for display. Three modules are deliberately UNWIRED from production: StockSageAllocationOptimizer (Frank-Wolfe MVO, ALLOC_BACKLOG #6 stretch), StockSagePyramid (scale-in opt-in calculator), and the CostProfile/portfolioCap APIs within StockSageKelly itself — all are tested but never called outside tests. StockSageLeverage and StockSagePartialLadder serve as display/backtester primitives without influencing live position sizes.

**Cross-domain edges:** IN: StockSageExpectedValue (ev() + cryptoRiskScaler + expectedWeeklyDollars — provides evR, winProbEstimate, rewardR that feed Kelly); StockSageNetEdge (defaultCosts + evaluate + clearsCost — the cost gate in allocate); StockSageRegime (adjustedWeight + MarketRegime.sizingBias — regime sizing bias applied to Kelly weight); StockSageVolRegime (StockSageIdea.volRegime?.sizingMultiplier — per-symbol vol brake applied after Kelly); StockSageCorrelationCluster (correlationAdjustedWeights — cluster de-weighting Step 2.5 in allocate); StockSageClusterCheck (check() — correlation gate in suggestAdd and rebalanceToEdge); StockSagePortfolioAnalytics (dailyReturns + correlationMatrix — feeds Optimizer Sigma and Allocator cluster step); StockSageRebalance (plan() — called by rebalanceToEdge after edge-weighting); StockSagePortfolioHeat (compute() — provides heatBefore for suggestAdd headroom); StockSageIndicators (annualizedVolatility — feeds Optimizer sigma vector); StockSageSector (sector() — passed as classify fn to StockSageAllocation.slices in MarketsRiskAllocationSection + MarketsView); StockSageConvictionCalibration (optional, passed through to StockSageExpectedValue.ev). OUT: StockSageCapitalAllocator.allocate() → MarketsView.swift (capital deployment plan displayed directly, not cached in store); StockSagePositionSizer.size() → StockSageTodayPlan, MarketsTodayActionsCard, MarketsView (all share count displays); StockSageAllocation.assetClass()/region()/breakdown() → MarketsRiskAllocationSection, MarketsView, StockSageStore (filtering), StockSageWhatIf, StockSageLiquidity, StockSageRiskFlags, StockSageExpectedValue; StockSageKelly.inputs() → StockSageJournal (backtest-implied Kelly from closed trades); StockSagePartialLadder.levels() → StockSageBacktester (scaleOutLadder exit mode simulation); StockSageLeverage.assess() → MarketsView (leveraged position display only).

### StockSageKelly.swift
**Purpose:** Pure fractional-Kelly calculator grounding the entire sizing domain: converts (winRate W, payoffRatio R) into per-position fractions and can scale a whole book of fractions to a portfolio heat ceiling. Contains the single authoritative constant maxFraction = 0.20 that every other sizing file imports.
**Key symbols:** KellyResult — the full f*/half/quarter/suggested fraction bundle; CostProfile — round-trip cost struct that subtracts from R before sizing; PortfolioKelly — uniform-scaled book result; StockSageKelly.compute() — core formula (f*=W−(1-W)/R, half-Kelly capped at maxFraction); StockSageKelly.portfolioCap() — uniform down-scale so ΣfractionsCap; StockSageKelly.maxFraction = 0.20 — the per-position hard ceiling shared across all engines; StockSageKelly.inputs() — derives (W, R) from backtest averages
**Inputs:** winRate (0–1 fraction), payoffRatio (avg-win÷avg-loss, dimensionless R), accountSize (dollars), optional CostProfile (commissionPct/slippagePct/bidAskPct all in %-space e.g. 0.1 = 0.1%). CostProfile.roundTripR divides by 100 to produce a fraction subtracted from R.
**Consumers:** StockSageCapitalAllocator.swift (allocate + suggestAdd + rebalanceToEdge — the primary pipeline caller); StockSageAllocationOptimizer.swift (compute per-position boxCap); StockSagePyramid.swift (uses maxFraction as riskCap default); StockSageJournal.swift (kellyInputs from backtest W/R); MarketsView.swift (stand-alone Kelly calculator UI, kellyWinRate text field → compute()); StockSageConvictionScaler.swift (references maxFraction for cap docs)
**Invariants:** suggestedFraction = min(halfKelly, 0.20) — NEVER exceeds 20% per position regardless of W and R. Non-positive edge clamps fStar to 0. CostProfile that exceeds R also floors to 0. portfolioCap() preserves per-position ranking by uniform (not selective) scaling.
**Gotchas:** CostProfile is defined and tested but NEVER passed by any production caller — all live calls use the nil default (costs=nil → costAdjustment=0, byte-identical to pre-cost results). Similarly portfolioCap() and PortfolioKelly are only called from tests — the production pipeline does its own heat cap inside StockSageCapitalAllocator.allocate, not via this primitive. KellyResult.dollarsToAllocate is capital under the Kelly model (≈ 20% of account), NOT the stop-risk dollars (~1% per trade) that StockSagePositionSizer sizes — maintainers often confuse these two.

### StockSagePositionSizer.swift
**Purpose:** Converts a stop-risk fraction into concrete whole shares and dollars-at-risk, making risk the input and size the output. The sole place in the engine that produces share counts.
**Key symbols:** PositionSize — shares (Int, floored down), dollarsAtRisk (Double), notional (Double), pctOfAccount (Double, 0–100 percent), riskPerShare (Double); StockSagePositionSizer.size() — the nil-or-result sizing function; StockSagePositionSizer.summaryLine() — one-line display string
**Inputs:** account (dollars > 0), riskFraction (0–1 fraction of account to risk, NOT a percentage), entry (dollars), stop (dollars). All must be finite and positive; entry==stop returns nil (undefined risk).
**Consumers:** StockSageCapitalAllocator.swift (every share count in allocate, suggestAdd, rebalanceToEdge flows through here); StockSageAllocationOptimizer.swift (produces AllocatedPosition shares); StockSageTodayPlan.swift (sizes rankedActions entries); MarketsTodayActionsCard.swift (displays share counts); MarketsView.swift (stand-alone sizer UI with rp/100 conversion, position-add calculator, pyramid calculator)
**Invariants:** .isFinite checked on ALL four inputs — a UserDefaults-persisted 'infinity' string would trap Int(Double.infinity), hard-crash, and persist. Int(exactly:) used instead of Int() to guard the 2^63 overflow where Double(Int.max) rounds UP and Int(raw) would trap. Shares always floor DOWN (never over-risk).
**Gotchas:** pctOfAccount is 0–100 PERCENT (e.g. 25.0), not a 0–1 fraction — callers in MarketsView multiply rp/100 before passing riskFraction, then display pctOfAccount * 1 (no further divide). riskFraction is always a 0–1 fraction going IN; the / 100 division happens at every MarketsView call site. A shares==0 result (account too small for even one share) is a valid return value distinct from nil — callers must guard both nil AND ps.shares>0.

### StockSageCapitalAllocator.swift
**Purpose:** Orchestrates the buy-side sizing pipeline: filters for positive-EV buy/strongBuy ideas, applies regime bias + vol-targeting + per-symbol vol-regime brake + correlation de-weighting, uniform-scales to the heat cap, then delegates share production to PositionSizer. Also provides suggestAdd (marginal add against an open book) and rebalanceToEdge (EV-weighted whole-book reweight via Rebalance.plan).
**Key symbols:** AllocatedPosition — per-position result (riskFraction 0–1, shares, dollarsAtRisk, notional, halfKelly pre-scale, evR); CapitalAllocation — full result (positions sorted desc by riskFraction, totalHeat 0–1 fraction, requestedHeat, scaleApplied, maxHeat); AddSuggestion — marginal add result (approved bool, heatBefore/heatHeadroom 0–1 fractions); EdgeRebalancePlan — wraps RebalancePlan + excludedSymbols + note; StockSageCapitalAllocator.allocate() — main pipeline; suggestAdd() — open-book marginal sizing; rebalanceToEdge() — EV-weighted rebalance; positionOrder(_:_:) — total-order position comparator (2026-07-03, exposed for tests)
**Inputs:** allocate(): ideas ([StockSageIdea] from store), account (dollars), maxHeat (0–1 fraction, default 0.08), optional calibration (StockSageConvictionCalibration), optional regime (MarketRegime). Internally calls StockSageExpectedValue.ev(), StockSageNetEdge (cost gate), StockSageKelly.compute(), StockSageRegime.adjustedWeight(), StockSageExpectedValue.cryptoRiskScaler(), idea.volRegime?.sizingMultiplier, StockSageCorrelationCluster.correlationAdjustedWeights(), StockSagePositionSizer.size().
**Consumers:** MarketsView.swift (calls allocate() directly at line 3477 as of 2026-07-03, not via store); StockSageStore.swift (calls allocate indirectly by providing ideas + regime; also uses assetClass() for filtering non-Index symbols)
**Invariants:** Cost gate (StockSageNetEdge.clearsCost) runs BEFORE Kelly sizing — no shares ever produced for a net-negative setup. Heat cap (maxHeat, default 0.08) is a hard ceiling on Σ riskFraction across all positions; uniform scale preserves per-position edge ranking. Per-position weight is suggestedFraction (half-Kelly capped at 20%), NOT raw halfKelly — prevents a lone idea from consuming 50% of account. rebalanceToEdge's per-position concentration cap (StockSageKelly.maxFraction=0.20) is applied iteratively and silently SKIPPED (with loud caveat) when only one fundable name exists, to avoid silently being undone by Rebalance.plan's own renormalization. positionOrder (2026-07-03, BUGHUNT #8): the Step-5 sort is a TOTAL order — riskFraction desc → symbol asc (raw `<`: "BTC" < "btc", case pairs never tie) → dollarsAtRisk desc → notional desc; Array.sorted(by:) is not guaranteed stable, so duplicate symbols tying on both original keys previously had unspecified relative order. Determinism only — every pre-existing distinct-symbol order is byte-identical.
**Gotchas:** allocate() contains a deliberately-noted limitation: the cost gate uses gross Kelly weights for sizing (net-payoff sizing deferred as a separate change). ## Domain: portfolio-risk

Eight stateless (enum/struct) modules that collectively enforce the "diversify by RISK, not by ticker count" mandate from MARKETS_INTELLIGENCE_RESEARCH.md §6. They form two functional groups: (1) retrospective risk analytics — StockSagePortfolioAnalytics computes the full Sharpe/Sortino/Calmar/VaR/correlation suite; StockSagePortfolioHeat measures live stop-level exposure; StockSageRiskParity derives inverse-vol rebalance targets; (2) pre-trade concentration guards — StockSageCorrelationCluster identifies hidden cliques of mutually-correlated names, StockSageCorrelationPrecheck classifies whether a candidate diversifies or concentrates the book, StockSageClusterCheck flags individual pairwise overlaps before entry, and StockSageRiskFlags aggregates all signals into a severity-sorted chip row. StockSagePortfolio is the sole persistent store, holding UserDefaults-backed holdings that every other module reads through StockSageStore.shared. All compute modules are pure and nonisolated; StockSageStore owns the async fetch-and-compute orchestration and exposes @Published properties the two Markets views bind to.

**Cross-domain edges:** IN: StockSageStore.refreshRiskParity/refreshPortfolioAnalytics/refreshPrecheck fetch via StockSageQuoteService → StockSagePriceHistory (closes + dates arrays) and use StockSageIndicators.annualizedVolatility to build RiskParityHolding. StockSageRiskFlags consumes TradeAdvice.Action (StockSageAdvisor), EarningsProximity (StockSageEarnings), LiquidityProfile.Tier (StockSageLiquidity), and StockSageAllocation.assetClass(). StockSageCorrelationCluster.correlationAdjustedWeights is called by StockSageCapitalAllocator (which also calls StockSagePortfolioHeat.compute to compute heatBefore/headroom). StockSagePortfolioAnalytics.dailyReturns/.percentile are reused by StockSageReturnShape, StockSageVolRegime, and StockSageExpectedValue.laneCorrelation. StockSageAllocationOptimizer calls dailyReturns + correlationMatrix. OUT: StockSageStore publishes analytics: PortfolioAnalytics?, correlation: CorrelationMatrix?, riskParity: [RiskParityTarget], precheck: [String: CorrelationPrecheck], portfolioBeta: Double? to MarketsView and MarketsRiskAllocationSection. StockSageCapitalAllocator consumes heat gating (heatBefore, headroom) and cluster-adjusted weights directly during allocate() and suggestNextTrade().

### StockSageCorrelationCluster.swift
**Purpose:** Detects the largest mutually-correlated clique of holdings (every pair >= 0.70) to name the hidden bet a heatmap makes visible, and applies correlation-adjusted 1/K de-weighting to capital allocation for cluster members.
**Key symbols:** CorrelationCluster (struct: symbols [String] sorted, minPairwise Double, note String), StockSageCorrelationCluster enum, threshold: Double = 0.70, largest(_:threshold:) -> CorrelationCluster?, correlationAdjustedWeights(symbols:weights:returns:threshold:) -> [Double]
**Inputs:** largest() takes a CorrelationMatrix (from StockSagePortfolioAnalytics.correlationMatrix). correlationAdjustedWeights() takes symbols, weights, and per-symbol daily return arrays (aligns to shortest common window internally).
**Consumers:** MarketsView line 1415 (portfolio heatmap section — calls largest on store.correlation to display cluster warning), MarketsRiskAllocationSection line 416 (same), StockSageCapitalAllocator lines 103, 288 (calls correlationAdjustedWeights before heat-scaling in both allocate() and suggestNextTrade())
**Invariants:** largest() returns nil if <3 holdings qualify (a 2-member cluster is a pair, not a clique). correlationAdjustedWeights() is a no-op (returns weights unchanged) if <3 symbols, if there are duplicate symbols, or if the shortest return series has <2 bars. **Gotchas:** The clique-finding algorithm is GREEDY (seed-expand), not optimal — it finds the largest clique reachable from each seed and takes the best, which can miss a larger clique that isn't reachable greedily from any single seed. ## Domain: exits-stops

The exits-stops domain is the risk-discipline and exit-mechanics layer of StockSage. It divides into three clusters: (1) price-based exits — StockSageTrailingStop (Chandelier ATR-ratchet, used in production for the idea detail sheet; recompute is tests-only) and StockSageRewardRisk (R:R ratio and break-even win-rate, wired into idea sorting, the detail sheet, TodayPlan, and the copy-plan export); (2) time and plan discipline — StockSageTimeStop (dead-money clock shown only when shouldExit=true in the open-positions list) and StockSageTradePlan (plain-text plan exporter, feeds into the copy-plan clipboard flow in MarketsView); (3) risk/ruin guardrails — StockSageTradeGate (pre-trade go/no-go gate embedded in TodayPlan and rendered in MarketsTodayActionsCard and MarketsView), StockSageDrawdown (historical underwater curve for the backtest card, 0-100% scale), StockSageDrawdownScenario (forward streak stress-test, 0-1 fraction scale — OPPOSITE of Drawdown), StockSageLossLimit (circuit breaker banner wired at the top of MarketsView, R-based policy hardcoded in the View), and StockSageMonteCarloRuin (bootstrap distribution of forward ruin using a seedable SplitMix64 PRNG for reproducibility, self-gated at 20 trades). All modules are pure/deterministic, Sendable, and have no side effects. The critical gotcha spanning the domain is a unit-scale split: UnderwaterCurve.maxDrawdown is 0-100 percent, while DrawdownScenario.drawdownPct and all MonteCarloRuin fields are 0-1 fractions — both are multiplied by 100 at MarketsView call sites but must never be confused in engine code.

**Cross-domain edges:** IN — StockSageIndicators.atr (feeds TrailingStop); StockSageJournal/TradeRecord.realizedR and .realizedProfit (feed LossLimit and MonteCarlo); StockSageJournal.equityRisk().maxConsecutiveLosses (feeds DrawdownScenario via ExpectedValue); StockSageAdvisor.TradeAdvice (stopPrice/targetPrice/conviction/regime/rationale/caveat feeds TradePlan and TradeGate and RewardRisk); StockSagePositionSizer.PositionSize (feeds TradePlan); StockSageNetEdge (feeds TradeGate rewardToRisk param with net cost-adjusted RR in MarketsView); StockSageStore (caches TrailingStop per symbol, publishes UnderwaterCurve after backtester history load). OUT — StockSageTodayPlan.TodayActionPlan.gate (TradeGateVerdict embedded as a field); StockSageExpectedValue.MoneyVelocitySummary (DrawdownScenario results as worstRunLosses/worstRunDrawdownPct); MarketsTodayActionsCard (consumes TradeGateVerdict.gateBadge); MarketsView (primary consumer of all 9 modules for display and clipboard export).

### StockSageMonteCarloRuin.swift
**Purpose:** Forward-looking tail-risk distribution: bootstraps the journal's closed-trade realized-R outcomes into 10,000 simulated futures at the configured risk-per-trade, reporting P(ruin), P(>20% drawdown), median max-DD, and 95th-pct max-DD. Self-gates below 20 R-defined trades.
**Key symbols:** SplitMix64 (seedable deterministic PRNG — makes simulation reproducible and unit-testable), MonteCarloRuin (struct: pRuin/p20DrawdownProb/medianMaxDD/p95MaxDD/sims/sampleSize), StockSageMonteCarloRuin.simulate (pure static), StockSageMonteCarloRuin.caveat (static String)
**Inputs:** trades: [TradeRecord] (only those with realizedR and !isOpen are used); riskFraction: Double (0-1); horizon: Int (default 100 trades); sims: Int (default 10,000); seed: UInt64 (default 0x5A1E_B0A7); ruinLevel: Double (default 0.5 = 50% of equity); minTrades: Int (default 20).
**Consumers:** MarketsView line 1586 (renders the forward-ruin text block in the risk section below the historical drawdown-streak line; colored warningSoft if pRuin>0.05; tooltip shows caveat string); StockSageMonteCarloRuinTests.
**Invariants:** Returns nil when fewer than minTrades R-defined closed trades exist — refuses to fabricate a scary or falsely comforting number from thin data. The guard floors at max(1, minTrades) (D-3 2026-07-03) so a degenerate minTrades <= 0 returns nil instead of trapping on modulo-by-zero. pRuin and p20DrawdownProb are 0-1 fractions; MarketsView multiplies by 100 for display. medianMaxDD/p95MaxDD are 0-1 fractions; MarketsView multiplies by 100. Equity is floored at 0 (equity < 0 → set to 0) to avoid negative compounding artifacts.
**Gotchas:** SplitMix64 is a custom struct (not Swift.RandomNumberGenerator's built-in) — same seed always yields identical output. MarketsView uses all defaults (no non-default params), so the simulation is always 10k sims at 50% ruin level over a 100-trade horizon. The caveat explicitly states the model UNDER-states clustered-loss tails (assumes independence, ignores regime shifts) — this must be surfaced to users and is shown via .help() tooltip. pRuin/p20DrawdownProb/medianMaxDD/p95MaxDD are ALL 0-1 fractions, NOT percents.

## Domain: ev-ranking

The ev-ranking domain converts conviction (0-1 signal-strength ordinal) and reward:risk into ranked, cost-honest, regime-gated opportunity boards powering the Markets tab. StockSageExpectedValue is the load-bearing hub: it owns every EV/velocity/fast-lane ranking function, the net-of-cost floor, and the summary card. ### StockSageCompoundingHorizon.swift
**Purpose:** Converts a weekly fractional return into a hypothetical time-to-double (or any target multiple) via log compounding. Deliberate UNWIRED engine-only module; ships tested but not surfaced in any UI or production call site.

## Domain: backtest-validation

The backtest-validation domain is the honesty layer of StockSage: it measures whether the advisor's fixed rules actually held up historically, and then aggressively discounts those measurements for the ways backtests lie. StockSageBacktester is the core simulation engine — a strict walk-forward sim with no look-ahead, conservative tie-breaks, and explicit cost accounting — which feeds both per-symbol UI panels and a conviction-calibration training set. StockSageStrategyBacktest aggregates per-symbol results into a strategy-wide verdict with a three-part significance gate and a Deflated Sharpe that applies a selection-bias haircut for the estimated 12 research iterations. StockSageDeflatedSharpe is a pure-math library (python-verified, all formulas from Bailey/López de Prado) that provides PSR and DSR; every nil-gate and the 0.95 'passes' threshold are deliberately chosen so the engine can never fabricate a confidence it hasn't earned. StockSageWhatIf is the odd-one-out: a pre-trade concentration projector (not a backtest tool) that lives in this domain only by file placement; it is UNWIRED from the Store, computed inline in MarketsView, and deliberately capped so a leveraged sizing figure doesn't produce a dishonest portfolio projection.

**Cross-domain edges:** FEEDS INTO: StockSageStore.refreshStrategyBacktest() calls StockSageBacktester.runDetailed() (feeding StockSageNetEdge.defaultCosts + a ^GSPC benchmark) and then StockSageStrategyBacktest.aggregate(), storing results in @Published strategyBacktest and backtestConvictionCalibration. StockSageStore also calls StockSageBacktester.run() (twice: allAtTarget vs chandelierTrail) for per-symbol backtest panels. MarketsView.swift reads store.backtest, store.backtestTrail, store.strategyBacktest for display and reads BacktestResult fields (winRate, avgWinR, avgLossR) to prefill the Kelly calculator. StockSageConvictionCalibration.fit(fromBacktest:) accepts [BacktestTrade] produced by runTrades — the conviction field on each trade is what calibration trains on. RECEIVES FROM: StockSageAdvisor.advise() (called bar-by-bar inside runTrades — the backtest measures the SAME rule as the live path). StockSageNetEdge.CostAssumption (round-trip bps). StockSagePartialLadder.levels() (for scaleOutLadder exit). StockSageIndicators.atr() (for chandelierTrail). StockSageAllocation.slices() and StockSageSector.sector() (consumed by StockSageWhatIf). The parity test (StockSageBacktestParityTests) enforces that the backtest decision path and the live buildIdeas path produce identical action/conviction at the final bar.

### StockSageBacktester.swift
**Purpose:** Walk-forward per-symbol backtester of the advisor's LONG rules — the honesty check on the Ideas feature. Also owns the ExitMode comparison infrastructure (allAtTarget, timeStop, chandelierTrail, scaleOutLadder) and the in-sample/out-of-sample walkForwardDecay helper.
**Key symbols:** BacktestTrade (one simulated trade with entry conviction), BacktestResult (aggregate metrics with significance gates), WalkForwardDecay (IS/OOS edge decay struct), ExitMode (enum: allAtTarget | timeStop | chandelierTrail | scaleOutLadder), StockSageBacktester.run() (main entry), runDetailed() (result+trades in one pass), runTrades() (raw trades, feeds calibration), summarize() (aggregation math, internal so it's unit-tested), simulateExit() (internal, unit-tested), scaleOutLadderExit() (internal, unit-tested), trailLevels() (chandelier ratchet, internal), walkForward() (fold-based stability check), foldRanges() (pure index math), walkForwardDecay() (extension, chronological IS/OOS split)
**Inputs:** StockSagePriceHistory (closes/opens/highs/lows/volumes/dates); StockSageAdvisor.advise() called bar-by-bar (no look-ahead); StockSageNetEdge.CostAssumption (optional round-trip bps); StockSagePriceHistory benchmark (optional, for relativeStrength); StockSagePartialLadder.levels() (for scaleOutLadder); StockSageIndicators.atr() (for trailLevels); StockSageDeflatedSharpe.moments() + probabilisticSharpe() (called inside summarize)
**Consumers:** StockSageStore.swift: runDetailed() at line 673 (strategy backtest, costs+benchmark fed), run() at lines 971-972 (single-symbol backtest, allAtTarget vs chandelierTrail comparison). StockSageStrategyBacktest.swift: aggregate() accepts [BacktestResult] and [BacktestTrade]. StockSageConvictionCalibration.swift: fit(fromBacktest:) accepts [BacktestTrade]. MarketsView.swift: reads store.backtest (BacktestResult), store.backtestTrail (BacktestResult), store.strategyBacktest (StrategyBacktest). Tests: StockSageBacktesterTests.swift, StockSageBacktestTests.swift, StockSageBacktestParityTests.swift.
**Invariants:** No look-ahead: bar i decision uses only closes[0...i]; entry fills at opens[i+1]. Stop wins ties (conservative worst-case when same bar touches both stop and target). One position at a time: cursor advances to exitIdx+1. Volumes forwarded only when history.volumes.count == n (alignment precondition, matches live advise() nil-gate). Benchmark date-aligned via forward-only bj pointer (O(n) total). costs==nil produces byte-for-byte identical output to the original cost-free path (nil → costPerShare=0 → r unchanged). isSignificant threshold is 20 trades. clearsMultipleTestingBar threshold is t>3.0 (Harvey-Liu-Zhu 2016, NOT the textbook 2.0). probabilisticSharpe is nil when <4 trades or zero variance — never fabricated. walkForwardDecay() is nil when fewer than 8 trades (not enough for a real 70/30 split). WalkForwardDecay.isRedFlag is gated on oosSignificant (>=20 OOS trades) to prevent false positives on thin slices.
**Gotchas:** BacktestResult.avgLossR is stored as a POSITIVE magnitude (the negation already applied in summarize()). BacktestResult.sharpe is per-trade (mean R ÷ stdev R), not annualized — callers must not compare it to an annualized Sharpe. openAtEndCount > 0 means avgR is optimistic (open trades close at last bar's close price, not at a real exit); MarketsView warns the user. Cost friction is charged against the PLANNED 1R risk (entry−stop), NOT against risk+cost — deliberate: dividing by risk+cost would make losers read as exactly −1R, hiding friction. chandelierTrail and scaleOutLadder are SIMULATED in the backtester (wired in ExitMode) but the chandelier trail levels come from trailLevels() which uses data through each bar without look-ahead. For chandelierTrail: the effective stop at bar j uses the PRIOR bar's ratcheted level (j-1), never the current bar. scaleOutLadder exit collapses multi-fill to a single blended-R so single-exit accounting still holds. walkForward() requires ALL parallel arrays (opens/highs/lows/dates/volumes) to be length-n; it returns [] silently on misalignment. BacktestTrade.conviction defaults to 0 so pre-conviction-era test constructions stay valid.

### StockSageStrategyBacktest.swift
**Purpose:** Aggregates per-symbol BacktestResults into a single honest strategy-wide verdict with pooled t-stat, skew/fat-tail-adjusted t-stat, Deflated Sharpe (PSR + selection-bias haircut), and pooled portfolio-proxy drawdown. Answers 'did these fixed rules hold up across many symbols' not 'did they work on one cherry-picked name'.
**Key symbols:** StrategyBacktest (aggregate struct), StockSageStrategyBacktest.aggregate() (entry point), sampleSymbols (24-symbol equity universe), estimatedStrategyTrials (12, the selection-bias DOF estimate), passesHonestSignificance (3-part gate), momentCorrectedTStat (Mertens/Lo IID non-normal SE), pooledDrawdownR (chronological equal-weight proxy DD), caveat (static survivorship-bias disclosure string)
**Inputs:** [BacktestResult] (per-symbol results); [BacktestTrade] (optional, pooled trades for t-stat/DSR/pooledDD); [Date] (optional trade entry dates 1:1 with trades for chronological DD sort); StockSageDeflatedSharpe.moments() + deflated() (for DSR computation)
**Consumers:** StockSageStore.swift: refreshStrategyBacktest() at line 683 calls aggregate(results, trades:, tradeEntryDates:) and stores into @Published strategyBacktest. MarketsView.swift: reads store.strategyBacktest at lines 3770-3792+ for the strategy backtest panel UI. StockSageConvictionCalibration.swift: comment references StockSageStore.refreshStrategyBacktest but does not call aggregate() directly. Tests: StockSageStrategyBacktestTests.swift.
**Invariants:** isSignificant requires >= 100 pooled trades (higher bar than per-symbol's 20). passesHonestSignificance requires all three: isSignificant AND tStat>3.0 AND (momentCorrectedTStat<=0 OR momentCorrectedTStat>3.0) — a raw-t pass that the fat-tail-adjusted t sinks does NOT get a PASS glyph. estimatedStrategyTrials (12) is a LOWER BOUND (conservative): under-counting trials makes the DSR bar easier, and since measured DSR is already ~0, raising it can only firm the 'unproven' verdict. Trials = estimated research iterations (not symbol count — researcher DOF != scan breadth). tradeEntryDates must be 1:1 aligned with trades for the chronological sort; misalignment falls back to received order silently. pooledDrawdownR is explicitly documented as EQUAL-WEIGHT + ignores concurrency/position sizing — it is a proxy, not a true sized-portfolio DD.
**Gotchas:** momentCorrectedTStat uses NON-excess kurtosis (normal=3, same as StockSageDeflatedSharpe.moments) — the formula is ((kurtosis-1)/4)*SR^2, not ((kurtosis-3)/4)*SR^2; this is the Mertens 2002 / Lo 2002 convention, verified against python reference. varTrialSharpe for DSR is the variance of per-symbol Sharpes (in-run cross-symbol dispersion), NOT the variance across research iterations — used as a defensible proxy. When varTrialSharpe=0 (degenerate), deflated() falls back to DSR==PSR rather than fabricating a haircut. survivorship bias is inherent (universe = currently-listed names only) — caveat string discloses this. sampleSymbols is EQUITIES ONLY (no FX/indices/crypto) because the long-side rules are built for equities.

### StockSageDeflatedSharpe.swift
**Purpose:** Pure-math library for Probabilistic Sharpe Ratio (PSR) and Deflated Sharpe Ratio (DSR) per Bailey and López de Prado. Corrects upward Sharpe bias from short/skewed/fat-tailed samples (PSR) and from scanning multiple strategies/symbols (DSR). All formulas python-verified before coding.
**Key symbols:** moments() (skew + NON-excess kurtosis, nil-gated at <4 pts or zero variance), probabilisticSharpe() (PSR = P(true Sharpe > benchmark)), expectedMaxSharpe() (selection-bias bar via Euler-Mascheroni / Gumbel), deflated() (full DSR = PSR vs expected-max bar), normalCDF() (via erf), inverseNormalCDF() (Acklam rational approximation), Result struct (psr, dsr, trials, passes: dsr>0.95)
**Inputs:** Arrays of per-trade R-multiples ([Double]) passed by callers; no external data dependencies — pure math library.
**Consumers:** StockSageBacktester.summarize(): calls moments() + probabilisticSharpe() to compute BacktestResult.probabilisticSharpe (PSR only, no selection-bias term). StockSageStrategyBacktest.aggregate(): calls moments() + deflated() to compute StrategyBacktest.deflatedSharpe (full DSR with trials=12 and varTrialSharpe from per-symbol Sharpe variance), and moments() for momentCorrectedTStat. MarketsView.swift: reads BacktestResult.probabilisticSharpe for the per-symbol panel (PSR label + PASS/BELOW BAR), and StrategyBacktest.deflatedSharpe for the strategy panel (DSR + passes flag). StockSageDeflatedSharpe.caveat string displayed as tooltip/a11y hint at lines 3845, 3941-3942. Tests: StockSageDeflatedSharpeTests.swift.
**Invariants:** moments() returns nil for < 4 points or zero variance — callers get nil, never a fabricated confidence. Result.passes is dsr > 0.95 (the honest 'real edge' bar). probabilisticSharpe() clamps n>=2 guard; returns 0 for <2 trades. inverseNormalCDF clamps p to (1e-12, 1-1e-12) to prevent boundary blowup. expectedMaxSharpe returns 0 for <=1 trial or V<=0 (no fabricated bar).
**Gotchas:** kurtosis is NON-EXCESS (normal distribution = 3, not 0) throughout. The PSR formula uses (kurtosis-1)/4 NOT (kurtosis-3)/4 — this is intentional, matching Mertens 2002 / Lo 2002 convention, and python-verified. Callers who pass kurtosis from moments() get the correct non-excess value; callers who pass a raw excess kurtosis will produce wrong results. DSR caveat (in static caveat string): assumes trials are roughly independent — a scan over correlated names violates this, making DSR itself optimistic; it corrects the direction of bias but not the exact size. Per-symbol PSR shown in the backtest panel deliberately OMITS the selection-bias term (trials haircut) — the caveat in MarketsView directs users to the strategy panel for DSR.

### StockSagePBO.swift
- **Purpose:** Probability of Backtest Overfitting via CSCV (Bailey–Borwein–López de Prado–Zhu 2017) — the selection-honesty sibling of `StockSageDeflatedSharpe`. Given N candidate configs' aligned return series, answers "how often does the in-sample winner land below median out-of-sample?" PBO ≈ 0.5 = no information; → 1 = the selection is overfit. Pure, Foundation-only, compiles standalone via swiftc (ablation runners chain it).
- **Key symbols:** `cscv(returns:blocks:)` → `Result?` (`pbo`, `medianLogit`, `splits`, `configs`, `blocks`, `blockLength`), `combinations(_:_:)`, `sharpe(_:)` (mean/sampleSD n−1; zero-variance → 0), `caveat`.
- **Inputs:** caller-supplied N×T return matrix (no data fetching). nil-guards: N<2, S<2 or odd, ragged rows, blockLength<2.
- **Consumers:** none in production — research/validation primitive (ablation runners feed it multi-config trial matrices; first demonstration 2026-07-07: the cap-vs-continuous arms, PBO 0.56–0.61). Tests: `StockSagePBOTests.swift` (7 hand-derived fixtures per spec-fidelity).
- **Invariants:** deliberately NO `passes` bar (no canonical PBO threshold exists, unlike DSR>0.95 — caveat says so); identical configs yield exactly 0.5 (mid-rank ties + half-weight on the exact-median atom — the load-bearing convention, pinned by test); ω = rank/(N+1) keeps every logit finite (no clamp); overfit weight compares rank vs (N+1)/2, never λ vs 0.0 (fp-safe).
- **Gotchas:** PBO measures overfitting of the SELECTION, not whether any config has true edge — N uniformly bad configs can still score ≈0. Hidden/unreported trials make it an UNDERestimate. Contiguous-block recombination assumes rough stationarity. C(S,S/2) grows fast (S=16 → 12,870 splits) — keep S modest.

### StockSageNetCostSim.swift
- **Purpose:** Net-of-cost simulation gate for candidate overlays (first client: the IRRX-cleaned reversal overlay, research roadmap item 3). Walk-forward rebalance simulator over caller-supplied return panels; charges per-side costs on one-way turnover (roundTripBps/2 per unit of Σ|Δw|); verdict via `StockSageDeflatedSharpe` (pass = DSR > 0.95 net).
- **Key symbols:** `Panel`, `rebalanceSeries(_:lookback:hold:roundTripBps:)` (nonisolated, pure), `irrxWeights` (industry-relative reversal, earnings-exclusion, look-ahead-free), `simulate(_:lookback:hold:roundTripBps:folds:embargo:trials:varTrialSharpe:)` → `SimResult` (`clearsNetOfCost`; the caller resolves the cost symbol via `StockSageNetEdge.defaultCosts(forSymbol:).roundTripBps`), `walkForwardFolds` with purge+embargo.
- **Inputs:** caller-supplied return panels (it does NOT fetch data); cost model from `StockSageNetEdge.defaultCosts(forSymbol:).roundTripBps`.
- **Consumers:** none in production yet — research/gate machinery (tests + future live-panel runs). Shipped 2026-07-03 from the Opus-lane O3 task after Fable review (review commit fixed a 2× cost double-count: full round-trip was charged per one-way turnover unit).
- **Invariants:** honesty floor — verdicts nil on thin/flat panels (never fabricated); synthetic-fixture caveat documented in the header (live-panel verdict pending); costs per-side on one-way turnover.
- **Gotchas:** turnover = Σ|Δw| counts each one-way trade — a round trip is 2 units; charging full roundTripBps per unit double-counts (~2×) and falsely refuses passing edges (the original pre-review bug). Terminal position not liquidated (understates cost by ≤ one exit). Fixture-based tests only; a future pass feeds real panels.

### StockSageCryptoHonesty.swift
**Purpose:** Crypto net-edge honesty engine (CRYPTO_RISK #2, 2026-07-03): runs the SAME real history through the EXISTING backtester three times — frictionless, at the tier's midpoint cost estimate, and at the high band — and surfaces the single most important honesty fact: did the edge flip from profitable to net-negative after costs. The Views fetch these rates via live quote fetches (USDSAR=X etc.) separately; this module is purely the conversion math.

## Domain: alerts-monitor

The alerts-monitor domain forms two loosely coupled subsystems over shared StockSageStore state. The passive in-app crossing layer (StockSageAlerts → StockSageStore.alerts [IdeaAlert]) fires on ideas-board refreshes whenever alertsEnabled is true, appending crossing events to a 50-item ring that MarketsView renders. The active push-notification layer (StockSageMonitor singleton) runs a 45-second polling loop, routes through three independent paths — strong-signal via StockSageSignalEngine, idea stop/target via StockSageAlertDecision (the only production caller, always with lastAlertedRecommendation:nil, restricted to stopBreach/targetHit by isPushableIdeaAlert), and user-set price levels via StockSagePriceAlertEngine — each with its own honesty gating (isSampleData, loadedFromCache, isStale guards). The three risk-characterization modules (StockSageReturnShape, StockSageVolStability, StockSageVolRegime) are advisory overlays wired purely into buildIdeas rationale notes; VolStability.sizingReliability is computed but deliberately unwired from sizing; VolRegime.sizingMultiplier is the single exception and is actively applied by StockSageCapitalAllocator (weight *= mult). StockSageGapRisk and StockSageLiquidity are consumed downstream (GapRisk by MarketsView idea-detail view; Liquidity by StockSageExpectedValue slippage modeling and StockSageRiskFlags) — neither affects the notification or crossing-detection paths.

**Cross-domain edges:** IN: StockSageStore.ideas ([StockSageIdea] with advice.action/stopPrice/targetPrice) → StockSageAlerts.detect; StockSageStore.priceAlerts ([PriceAlert]) → StockSageMonitor.checkPriceAlerts; StockSageStore.{isSampleData, loadedFromCache, userSymbols, fetchAllSymbols()} → StockSageMonitor honesty gates; StockSageSignalEngine.generateSignal(for:) → StockSageMonitor.runCycle (strong-signal path); StockSageQuoteService.fetchQuotes(for:) → StockSageMonitor (watchlist/price-alert paths); MemoryManager.shared.concurrencyLimit() → StockSageMonitor thermal throttle; StockSageIndicators.annualizedVolatility → VolRegime + VolStability; StockSagePortfolioAnalytics.{dailyReturns, percentile} → ReturnShape + VolRegime; StockSageAllocation.{assetClass, region} → StockSageLiquidity.isUSDPriced. OUT: StockSageStore.alerts ([IdeaAlert]) → MarketsView rendering; StockSageMonitor → UNUserNotificationCenter (push notifications); StockSageIdea.volRegime (VolRegime?) → StockSageCapitalAllocator (weight *= sizingMultiplier); LiquidityProfile → StockSageExpectedValue (slippage cost), StockSageRiskFlags (.thin caution flag); GapRiskScenario → MarketsView idea-detail inline display."

### StockSageAlerts.swift
**Purpose:** Pure crossing detector that converts two consecutive [StockSageIdea] snapshots (previous poll → current poll) into discrete IdeaAlert events. Fires only on state changes, never on standing conditions, so no external dedup set is needed.
**Key symbols:** IdeaAlert (Sendable, Equatable, Identifiable event struct with UUID id/symbol/kind/detail/price), IdeaAlert.Kind (flipBullish/flipBearish/stopBreach/targetHit), StockSageAlerts.detect (static nonisolated pure crossing detector), isWarning (Bool property: stopBreach or flipBearish)
**Inputs:** [StockSageIdea] previous and current snapshots; reads advice.action, price, advice.stopPrice, advice.targetPrice from each idea
**Consumers:** StockSageStore.swift (line 333: detect called inside buildIdeas when alertsEnabled is true, results prepended to store.alerts capped at 50); MarketsView.swift (line 3229: alertIcon(_:) renders IdeaAlert.Kind visually); StockSageAlertDecision.swift and StockSageMonitor.swift reference the same crossing logic in comments; StockSageAlertsTests.swift, StockSageMonitorTests.swift, StockSageTestFixtures.swift in tests
**Invariants:** New symbols (no previous entry) NEVER fire on first appearance — the prevBy lookup simply yields no match. ### StockSageAlertDecision.swift
**Purpose:** Stateless pure decision function for the monitor's push-notification path: given a symbol's current and prior state, returns at most one StockSageAlert (the single most-actionable event) or nil to stay silent. Dedupes strong-signal alerts via lastAlertedRecommendation.
**Key symbols:** StockSageAlert (symbol/kind/reason struct), StockSageAlert.Kind (newStrongBuy/newStrongSell/flip/stopBreach/targetHit), StockSageAlertDecision.evaluate (static nonisolated priority-ordered decision: stop > target > strong-signal, with dedup)
**Inputs:** symbol, recommendation (StockSageRecommendation), price, priorPrice (both Doubles from live quote), stop/target (optional Doubles), lastAlertedRecommendation (optional, for dedup)
**Consumers:** StockSageMonitor.swift checkIdeaAlerts (line 236) is the ONLY production caller; always passes lastAlertedRecommendation: nil (see gotchas); StockSageAlertDecisionTests.swift in tests
**Invariants:** Priority order is hard-coded: stop breach checked first, then target hit, then strong-signal. Only .strongBuy and .strongSell trigger signal alerts; .buy/.sell/.hold are silent. **Gotchas:** StockSageMonitor always passes lastAlertedRecommendation: nil to evaluate, which means the 'new strong signal' and 'flip' branches can never suppress themselves via dedup from this call site. ### StockSageMonitor.swift

Direct (non-loop) callers run in a non-cancelled task, so their behavior is byte-identical.

**Invariants:** sizingMultiplier is the MINIMUM of absoluteBrake and percentileBrake, hard-floored at 0.25 — never reaches 0. absoluteBrake = min(1, medianVol/currentVol) — brakes as current vol rises above median. percentileBrake = max(0.5, 1 − max(0, pct−0.5)) — linear from 1× at p≤0.5 to 0.5× at p=1.0. Takes the more conservative (min) of the two. Requires ≥5 valid rolling windows (degenerate guard). Rolling window anchors use inclusive end-index (i...i not i..<i) — a half-open range would silently lag every reading by one day.
**Gotchas:** The absolute-vol anchor (medianVol/currentVol) corrects for a known percentile-of-own-history blind spot: a structurally high-vol name (crypto) can read 'calm' at absolutely dangerous levels — the absolute brake fires independently of the percentile. The leverage effect (Engle 1982 ARCH; Black 1976) is cited as the research basis, with an honest caveat: it is strongest for equities/credit and weaker for FX/commodities. sizingMultiplier < 0.95 is the note-display threshold in buildIdeas — values between 0.95 and 1.0 are computed but produce no user-visible output. The VolRegime struct is stored on StockSageIdea.volRegime (VolRegime?) so the allocator can read it without re-computing.
