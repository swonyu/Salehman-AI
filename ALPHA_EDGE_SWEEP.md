### ✅ DONE #1 [earnings-rank] — Earnings-gated EV/velocity rank: fold imminent binary-event risk into the ranking key (real fetched dates only)
**mechanism:** Adds a NEW disjoint demotion band to the existing rank keys in StockSageExpectedValue.swift, keyed off the SAME real EarningsProximity the Store already fetches and caches (StockSageStore.earnings, line 509, populated only from Yahoo calendarEvents via StockSageEarnings.fetchNextEarnings). A pure helper earningsRankPenalty(severity:) returns 0 for .clear/.soon/nil (unknown is never penalized — absence is labeled, not assumed dangerous) and 2000 for .imminent (≤3 days). New OVERLOADS rankByEV(_:regime:earnings:) and rankByVelocity(_:holds:earnings:) look up earnings[idea.symbol.uppercased()] and subtract the penalty inside the existing rank-key comparison. The 2000 constant sits ABOVE the conviction band (−1000, line 128/169) and the max possible base EV (verified: rewardR cap 50, winProb 0.35+conv·0.23 ⇒ maxEV ≈ 28.58), yet BELOW the cost (−500_000, line 129/168) and regime (−1_000_000, line 156) bands — so an imminent-earnings idea always sinks below every clean same-or-lower-EV peer while ordering among non-imminent ideas stays byte-identical. Displayed EV/velocity is unchanged; only the rank key shifts (matches the existing contract at lines 105/135).

**signature:** nonisolated static func earningsRankPenalty(_ severity: EarningsProximity.Severity?) -> Double  // .imminent ? 2000 : 0
nonisolated static func rankByEV(_ ideas: [StockSageIdea], regime: MarketRegime? = nil, earnings: [String: EarningsProximity]) -> [StockSageIdea]
nonisolated static func rankByVelocity(_ ideas: [StockSageIdea], holds: VelocityHoldDays = .defaults, earnings: [String: EarningsProximity]) -> [StockSageIdea]

**test:** Two equity buy ideas with IDENTICAL base velocity keys (same conv 0.80, entry 100, stop 98, target 110, hold 12); A reports in 2 days (.imminent), B in 30 (.clear). With empty earnings, order is original (A,B); with earnings-gating, .imminent A sinks below B ⇒ (B,A). Band invariant asserted: 1000 + maxEV(~28.58) < 2000 < 500_000 < 1_000_000. Penalty(None/.soon/.clear)==0, penalty(.imminent)==2000. Python model reproduces the exact key math (qAdjEV = evR·(0.4+0.6·conv), velocityKey = qAdjEV/hold) and all asserts pass.

**caveat:** Demotes ONLY on a real fetched earnings date (Yahoo calendarEvents, equities only). A symbol with no entry in the earnings map is UNKNOWN and is NOT penalized — a stock that actually reports tomorrow but whose date Yahoo declined will rank un-demoted, so the existing per-idea EarningsProximity.note (StockSageEarnings.swift:25) stays the load-bearing disclosure. The penalty re-orders WITHIN the clean-idea band only — it cannot rescue a cost-failed or regime-banned idea, and never changes displayed EV. It is a de-prioritization, not a filter: an imminent-earnings idea can still surface if it is the only candidate. .soon (≤10d) is deliberately un-penalized.

**edgeRationale:** A protective stop is an intraday promise an overnight earnings gap opens straight through — the exact risk StockSageEarnings was built to flag. Ranking an idea #1 the night before it reports puts the owner's largest sized position into the one setup where his stop is least likely to hold, turning a planned −1R into an un-modeled multi-R gap loss. Folding already-fetched earnings proximity into the rank means capital flows to setups whose stop can actually bound the loss.

### ⬜ #2 [volume-confirmation] — Real-volume breakout confirmation verdict (donchian breakout × volume participation)
**mechanism:** New pure function in StockSageIndicators (after isBreakout, which ends at line 221) that orchestrates THREE already-tested primitives: donchian (line 209), isBreakout (line 219), volumeConfirmation (line 142). Builds the Donchian channel on bars EXCLUDING the current one (dropLast — the channel comment at lines 207-208 already mandates this look-ahead discipline), tests the current close with isBreakout, and only then consults volumeConfirmation. Returns a 4-state verdict so the caller reacts honestly to each case rather than collapsing 'breakout with no volume data' into 'confirmed'. REAL VOLUME ONLY: volumeConfirmation already returns nil when volume is absent/all-zero (priorAvg>0 guard, line 151) — that nil surfaces as the explicit .volumeAbsent case, never silent confirmation.

**signature:** enum BreakoutVerdict: Sendable, Equatable { case confirmed(volRatio: Double); case unconfirmed(volRatio: Double); case noBreakout; case volumeAbsent }
nonisolated static func breakoutConfirmation(closes: [Double], highs: [Double], lows: [Double], volumes: [Double], period: Int = 20) -> BreakoutVerdict?  // nil when closes.count < period+1 or lengths mismatch

**test:** 25-bar series; channel uses the first 24 bars (dropLast), period 20. (1) Confirmed: flat 100 for bars 0..23 ⇒ upper=100, bar 24 close=105 (>100, strict), volumes 1000 prior / 5000 last-3 ⇒ ratio 5.0 ⇒ .confirmed(5.0). (2) Unconfirmed: same prices, volumes 1000 prior / 200 last-3 ⇒ ratio 0.2 ⇒ .unconfirmed(0.2). (3) noBreakout: bar 24 close=100 == upper (isBreakout is strict '>') ⇒ .noBreakout. (4) volumeAbsent: all-zero volumes ⇒ volumeConfirmation nil ⇒ .volumeAbsent. Each branch's number reproduced by hand: max(), strict '>', recentAvg/priorAvg.

**caveat:** A confirmed breakout is a higher-quality ENTRY, not a probability of profit — above-average volume only says the move had real participation; breakouts still fail. .volumeAbsent is an honest 'cannot confirm' (FX/indices have no share volume), NOT a soft yes. volumeConfirmation compares the last 3 bars to the prior 20, so a stale/partial-day volume bar can momentarily mislabel a real breakout as unconfirmed; calibrated for end-of-day daily bars. period defaults to 20 (~1 month); a 52-week breakout needs period 252 and ≥253 bars.

**edgeRationale:** Buying breakouts on thin volume is the classic way retail gives money back — the false breakout reverses and stops you out. Gating the entry on REAL volume participation downgrades exactly those traps before capital is committed, and the explicit .volumeAbsent state stops the app pretending an FX/index 'breakout' is volume-confirmed when no volume exists to confirm it.

nonisolated static func earningsRankFlag(for idea: StockSageIdea, earnings: [String: EarningsProximity]) -> EarningsRankFlag

**test:** earn = {AAA:2, BBB:7, CCC:30}. rank_flag('aaa') == .demoted(2) (case-insensitive, like .uppercased()); 'BBB' == .approaching(7); 'CCC' == .clear(30); 'ZZZ' == .unknown. **edgeRationale:** When the owner's strongest-signal idea drops off the top of the board, an unexplained shift invites an override ('why isn't my best setup #1? — I'll buy it anyway'), re-introducing the exact gap risk the rank just removed. ### ⬜ #4 [regime-playbook] — Regime playbook — name the strategy that fits the current tape

No new data, no fetch — re-reads the regime the store already computed. Same switch shape as the existing regime-composer at StockSageExpectedValue.swift:144-152.

**test:** Pure mapping: playbook(.trendingBull)==.momentum, playbook(.trendingBear)==.defensive, playbook(.ranging)==.meanReversion, playbook(.crisis)==.capitalPreservation. Swift XCTest mirror builds a MarketRegime via StockSageRegime.assess — clean uptrend + vix 15 ⇒ .trendingBull ⇒ .momentum; crash series + vix 50 ⇒ .crisis ⇒ .capitalPreservation; flat/oscillating ⇒ .ranging ⇒ .meanReversion. **caveat:** A regime is a sizing/strategy BIAS, not a forecast — it says which approach historically fits the tape, not what the market will do next. ### ⬜ #5 [regime-playbook] — Does this idea fit the regime playbook? — flag off-playbook trades

Spec 4 names the regime's playbook; this joins them: a pure boolean answering 'is this idea's stance ALIGNED with what the regime favors?'. **signature:** nonisolated static func fitsPlaybook(action: TradeAdvice.Action, regime: MarketRegime?) -> Bool?  // nil when regime unknown

**test:** trendingBull: fits(.buy)==true, fits(.sell)==false. trendingBear: fits(.reduce)==true, fits(.strongBuy)==false. crisis: fits(.strongBuy)==false, fits(.avoid)==true. ranging: fits(.buy)==true and fits(.sell)==true. nil regime ⇒ nil. **caveat:** An alignment FLAG, not a veto — a fundamentally-right contrarian trade can be off-playbook and still work; the advisor's own signal (which already vetoes longs in a name's 12-1 downtrend) is the primary read. **edgeRationale:** The per-idea complement to Spec 4's headline. ### ⬜ #6 [vol-regime-breakout] — atrExpansion — volatility-regime ratio (fast ATR ÷ slow ATR)
**mechanism:** New pure indicator in StockSageIndicators (alongside atr at line 84, efficiencyRatio at line 106). Calls the EXISTING Wilder atr() twice over the SAME real OHLC arrays — short period (default 14) and long (default 50) — and returns fast÷slow. >1 means recent true range is wider than its longer-run baseline (volatility EXPANDING); <1 means contracting (chop/coiling). Computed only from real highs/lows/closes; returns nil if either ATR is unavailable or the slow ATR is 0; never fabricates a value. Reuses the audited atr() so there is no second ATR implementation to drift.

**signature:** nonisolated static func atrExpansion(highs: [Double], lows: [Double], closes: [Double], fastPeriod: Int = 14, slowPeriod: Int = 50) -> Double?

**test:** Re-implements Wilder ATR identically in Python. Flat alternating series (base 100±0.2, ranges 0.6) over 70 bars ⇒ expansion ≈ 1.0 (asserted <1.10 chop floor). Same 60 flat bars + one wide bar (H=106,L=100.2,C=105.5) ⇒ expansion ≈ 1.38 (>1.10). Insufficient bars (10) ⇒ None. (Standalone Swift printed 2.08 for a stronger expansion fixture.)

**caveat:** A ratio is a RELATIVE volatility read, not a direction or forecast — expansion can precede a breakout OR a violent reversal; pair it with a price trigger (Spec 2) and never read it as bullish on its own. Needs slowPeriod+1 real bars (~51 days at default) or returns nil. This is a standalone, additive indicator: it adds no parameter to any existing call site, but nothing consumes it yet — its money-edge only lands once a caller gates breakout entries on it, which is why it ranks below the verdict that already orchestrates a usable signal.

**edgeRationale:** It tells you WHEN the tape is actually moving vs dead. Donchian breakouts taken in a quiet, contracting market are the classic false-breakout bleed (many tiny stop-outs compounding into a losing month). Requiring fast>slow ATR before acting concentrates breakout attempts into sessions where range is genuinely expanding, cutting the count of noise trades that each cost ~1R.

### ⬜ #7 [confluence-score] — StockSageConfluence — transparent 0–N count of INDEPENDENT bullish confirmations
**mechanism:** A new pure namespace that calls FIVE confirmations that already exist, each as one yes/no vote, returning BOTH the count and labeled per-component verdicts (fired/failed/unavailable). The reads: trendOK (line 201), returnOverPeriod>0 (line 131), relativeStrength>0 (line 162), MarketRegime.riskScore>0 (Regime), volumeConfirmation?.confirmed (line 142). CRITICAL HONESTY RULE — each component is TRI-STATE (Bool?): nil ⇒ real data absent (FX/index volume, <253 bars, no benchmark, no regime) ⇒ reported .unavailable and EXCLUDED from BOTH numerator and denominator. score = count true; outOf = count non-nil. The score is literally a sum of existing booleans — no new market math, no new thresholds.

**signature:** struct Result { let score: Int; let outOf: Int; let components: [Component]; let caveat: String; var fraction: Double? }
nonisolated static func score(closes: [Double], volumes: [Double]? = nil, benchmarkCloses: [Double]? = nil, regimeRiskScore: Double? = nil, momentumPeriod: Int = 126) -> Result

**test:** 300 monotonically-rising bars + rising volume + flat benchmark + risk-on regime ⇒ (score,outOf)==(5,5). Drop volume/benchmark/regime ⇒ (2,2) — only trend+momentum had real data, both pass, absent inputs shrink the denominator not the score. 60-bar history (trend unavailable, others absent) ⇒ (1,1) momentum only. Python mirrors the tri-state arithmetic and the trend_ok/vol_conf guards; all asserts pass.

**caveat:** The components are NOT statistically independent — trend (12-1 TSMOM), 6-month momentum and relative strength are all trailing-return constructions and move together, so 4/4 or 5/5 means broad AGREEMENT, not orthogonal edges, and must not be read as a probability of profit. **edgeRationale:** Today the advisor collapses everything into one conviction scalar; the owner can't quickly see HOW MANY independent things agree. A 0–N count with per-component pass/fail/absent labels lets him rank competing ideas by BREADTH of confirmation and skip lone-signal traps (price popped but RS failing and volume absent ⇒ 1/2, stand aside) — turning the black-box conviction into an auditable checklist.
