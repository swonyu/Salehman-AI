## 1. Goal (one sentence)

From the open idea detail sheet, the user can step to the previous/next idea **in the exact
order the board currently shows** (chevrons + "N of M" in the sheet header, ⌘↑/⌘↓), without
closing the sheet and without rapid stepping firing the sheet's six per-symbol store
refreshes for every symbol skipped over.

## 2. Design constraints (adversarially established — respect ALL)

1. Chevron `Button`s + "N of M" position label live in the **sheet header next to the X
   close button** (header HStack at 4417–4435).
2. Stepping walks **`displayedIdeas`** (MarketsView.swift:2829) — the SAME post-sort/filter/
   search order as the board. Never `store.ideas` (that's pre-sort/filter — position lies).
3. The current index is resolved **by `idea.id` AT PRESS TIME** — `displayedIdeas` mutates
   under background refresh (`store.refresh()` / `refreshIdeas()` from the `.task` at 204,
   monitor-driven store updates), so a render-time index can be stale when the press lands.
   Clamp at the ends: disable the chevrons, no wrap-around.
4. Shortcuts are **⌘↑ / ⌘↓ — NOT bare arrows, NOT j/k**. The sheet hosts live TextFields
   (`journalField` → the sizer's "Acct $"/"Risk %" fields, def at 1915, called at 4202/4204
   inside `positionSizerPanel` which the sheet renders at 4511/4522). A bare `.upArrow` key
   equivalent steals the responder chain's cursor keys; a `j`/`k` equivalent steals typing.
   House pattern: the X button binds `.keyboardShortcut(.cancelAction)` (4434); CodeView
   binds only ⌘-modified equivalents (CodeView.swift:710–724).
5. Setting `selectedIdea` to the neighbor updates the **item-bound sheet in place**
   (`.sheet(item: $selectedIdea)` at 218 — no dismiss/re-present) and `.task(id: idea.symbol)`
   (5143) re-fires per symbol. That task runs **SIX sequential store refreshes** (MTF,
   precheck, earnings, seasonality, liquidity, trailing stop), so rapid stepping MUST be
   debounced: sleep ~300 ms first, bail if cancelled (the `.task(id:)` identity change
   cancels the old task), and check `Task.isCancelled` between refreshes — extending the
   existing F31 `mtfFetchCompleted` guard pattern already at 5146–5151.
6. Honesty floor: the "N of M" label is shown ONLY when the shown idea actually resolves in
   `displayedIdeas`. Opened from `bestOpportunityCTA` (3527) or the alerts panel (3880) the
   idea can be filtered off the board — then both chevrons disable and NO label renders.
   Never fabricate a position.

```swift
// MARK: - Detail-sheet prev/next candidate navigation (pure index math)

/// Pure resolution for the ideas detail-sheet prev/next stepper. Kept top-level (internal,
/// NOT nested private inside MarketsView) so `Salehman AITests` reaches it via
/// `@testable import Salehman_AI`.
///
/// Board order = `displayedIdeas` (post sort/filter/search) — the SAME order the user sees.
/// The board mutates under background refresh, so callers pass a FRESH `ids` snapshot and
/// re-resolve at press time. nil means "cannot step": unknown id (the shown idea fell off
/// the board) or past either end. No wrap-around — clamping is the contract; the UI
/// disables the chevron.
enum SheetCandidateNavigation {
    /// 0-based index of the neighbor `delta` steps from `currentID` in `ids`, or nil
    /// (unknown id / out of range).
    static func neighborIndex(ids: [String], currentID: String, delta: Int) -> Int? {
        guard let i = ids.firstIndex(of: currentID) else { return nil }
        let j = i + delta
        return ids.indices.contains(j) ? j : nil
    }

    /// 1-based "N of M" position label, or nil when `currentID` is not on the board —
    /// honesty floor: never fabricate a position for an idea the board no longer shows.
    static func positionLabel(ids: [String], currentID: String) -> String? {
        guard let i = ids.firstIndex(of: currentID) else { return nil }
        return "\(i + 1) of \(ids.count)"
    }
}
```

```swift
    /// the same post-sort/filter order the board renders). The current index is re-resolved
    /// by id HERE, AT PRESS TIME: displayedIdeas mutates under background refresh, so an

    private func stepSheet(_ delta: Int, from id: String) {
```

### Step 7 — Debounce the six-refresh `.task` for rapid stepping

Constraint 5 mechanics against the real code at 5143–5157: `.task(id: idea.symbol)` re-fires
for EVERY symbol stepped through, and its body runs SIX sequential awaits
(`refreshMultiTimeframe` → `refreshPrecheck` → `refreshEarnings` → `refreshSeasonality` →
`refreshLiquidity` → `refreshTrailingStop`). `.task(id:)` cancels the outgoing task on an
identity change, but (a) the fetches have already been ISSUED if the task got past the top,
and (b) a cancelled task keeps executing past each `await` unless it checks. Design: sleep
300 ms at the top (a skipped-over symbol never starts fetching — its task is cancelled
during the sleep), then extend the existing F31 `Task.isCancelled` guard (5146–5151) to
between EVERY refresh so a mid-chain step/dismiss stops the remaining fetches for the old
symbol. Cost: +300 ms before fetches start on any open; the `mtfFetchCompleted`-bounded
spinner (4597) already covers that window. `--qa` bails before the sleep — QA snapshots
stay deterministic and offline.

**File:** `Salehman AI/Views/MarketsView.swift` · **Anchor:** the `.task` block at
5143–5157 (unique — PF-4).

Edit — **OLD (exact):**
```swift

            // mid-fetch the task is cancelled but execution continues past the await — inserting
            // into mtfFetchCompleted in that case would cause the NEXT open to skip the spinner
            // and flash "Weekly timeframe unavailable" while a new fetch is in flight.
```

```swift
            // into mtfFetchCompleted in that case would cause the NEXT open to skip the spinner
```
