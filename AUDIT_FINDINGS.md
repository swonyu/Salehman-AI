### ✅ DONE #2 [low] Long-side ATR stop can return negative for very-high-ATR names — no positive-floor guard (short side already guards its target)
**fix:** In stopTarget's buy branch, `let s = price - dist` is returned unguarded; when `dist = mult*atr` exceeds `price` (ATR > ~40% of price, e.g. thin micro-cap / crypto bar, made marginally more reachable by the new 2.5x multiplier), `s` is negative — nonsensical to display, and asymmetric with the short side which already drops a degenerate target via `t > 0 ? t : nil` (line 243). Floor the long stop: `let s = Swift.max(price - dist, price * 0.001)` (and derive the target from the floored s so the 2:1 stays consistent), OR return (nil,nil) when `dist >= price` to drop an untradeable plan. Mirror the defensive intent already present on the short side.

### ⬜ #5 [low] Scale-out ladder charges round-trip friction ONCE on the collapsed single fill — undercounts a real N-rung exit's per-chunk cost (ladder net-R not apples-to-apples with allAtTarget)
**fix:** scaleOutLadderExit collapses the multi-fill ladder into one blended-R fill (simulateExit:145), and run() then applies round-trip friction ONCE on that single notional (line 118-119: costPerShare = roundTripBps/10000*entry). A genuine N-rung scale-out pays round-trip friction on N separate exit fills plus the entry; the collapse charges entry+1 exit, OVERSTATING ladder net R by ~(N-1) exit legs of bps. The reconstruction itself round-trips exactly (no arithmetic bug), so this is a modeling undercount, not a crash. Two acceptable fixes: (a) charge exit friction per banked rung inside scaleOutLadderExit — accumulate `realized -= fraction * costRate * rungExitPrice / risk` as each rung fills, plus on the stop/end remainder, instead of one cost on the blended fill; or (b) document in the function header (137-139) that ladder net-R omits the extra per-chunk exit friction so it is NOT read as apples-to-apples against allAtTarget's single exit. Prefer (a) for fidelity since the audit's other ladder invariants are exact.
